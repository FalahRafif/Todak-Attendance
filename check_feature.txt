# Check Feature KlikAbsen

Tanggal audit: 2026-06-10
Metode audit: baca BRD, README, revision_feature, route/controller/service/view/model yang relevan, dan diff implementasi terakhir. Status ini adalah analisis kode statis + hasil verifikasi syntax/cache, bukan QA manual penuh di browser.

## Ringkasan Eksekutif

### Sudah ada dan relatif berfungsi
- Auth login/logout dan redirect role Admin, HRD, Employee.
- Role-based access untuk prefix Admin, HRD, Employee.
- Admin CRUD master utama: departments, positions, work locations, shifts, holidays, employees.
- Employee dashboard, absensi hari ini, check-in, check-out, riwayat, kalender, pengajuan izin/cuti, koreksi absensi, profil.
- HRD dashboard, monitoring absensi, belum check-in, belum check-out, terlambat, outside radius, approval pengajuan, approval koreksi, laporan harian/bulanan, export CSV, jadwal karyawan read-only, saldo cuti read-only.
- Attachment preview signed URL untuk profile image terenkripsi.
- Foto attendance sekarang punya relasi `checkInPhoto` dan `checkOutPhoto`, dan HRD detail menampilkan foto via signed preview URL.

### Sudah direvisi pada implementasi terakhir
- Outside radius pada check-in/check-out sekarang wajib `work_mode_id` + `note` di backend.
- Outside radius menolak work mode `office` dan memastikan reference group `WORK_MODE`.
- Check-in terlambat sekarang mengubah `status_id` menjadi `late` jika melewati toleransi shift.
- `late_minutes` sekarang dihitung setelah toleransi shift.
- Check-out Employee sekarang menghitung `early_leave_minutes` dari shift end, termasuk overnight shift sederhana.
- HRD Belum Check-in sekarang exclude holiday, employee day off, dan approved leave berdasarkan range `leave_requests`.
- Employee leave request sekarang validasi overlap pending/approved.
- Employee annual leave sekarang validasi sisa `leave_balances` saat submit.
- Leave cancel dan correction cancel sekarang update status `cancelled`, bukan delete row.
- Employee correction sekarang menolak duplikasi pending untuk employee + tanggal sama.
- HRD leave approval sekarang membuat `leave_request_details` per tanggal.
- HRD leave approval sekarang mengurangi `leave_balances` untuk annual leave.
- HRD correction approval sekarang menghitung ulang `late_minutes`, `total_work_minutes`, `early_leave_minutes`, dan status present/late.
- Controller HRD leave/correction approval menangkap RuntimeException agar error bisnis tampil ke UI, bukan 500.

### Sudah ada tapi belum berfungsi penuh / flow masih kurang
- Activity log table/model ada, tapi aksi penting belum konsisten dicatat ke `activity_logs`.
- `approvals` table/model ada, tapi approval leave/correction/outside radius masih langsung update tabel target, belum memakai tabel `approvals`.
- Leave/correction attachment column ada, tapi form Employee belum upload attachment pendukung.
- Attendance photo masih disimpan di public storage, walau preview HRD sudah lewat signed route.
- Employee attendance detail sudah load relasi foto, tetapi view Employee detail belum terlihat menampilkan foto selfie.
- Validasi outside radius di frontend belum jelas disable submit sampai work mode + note terisi; backend sudah wajib.
- Employee leave submit/correction submit belum catch RuntimeException di controller store, sehingga validasi service seperti saldo/overlap berisiko 500 jika tidak ditangani global.
- HRD daily/monthly report ada, tapi kolom/filter belum selengkap BRD.
- HRD leave requests dan attendance corrections belum punya filter status/tanggal/departemen.
- HRD employee schedules dan leave balances read-only saja; tidak ada create/update dari Admin/HRD.
- Admin employee detail route target BRD belum ada; hanya index/create/edit/update/delete.
- Sidebar/menu wording masih sebagian English di config, walau view utama sudah banyak dibenahi.
- Monthly summary generate masih memakai `updateOrCreate` dengan `uuid` di update payload, sehingga UUID existing masih berpotensi berubah tiap generate.
- Leave approval sudah memakai beberapa operasi multi-table, tetapi belum dibungkus `DB::transaction` di `HrdService`.
- Leave balance reduce belum idempotent jika approve dipanggil ulang secara race condition; status pending dicek, tetapi tetap perlu transaction/locking untuk aman.

### Belum ada
- Admin Settings, References, Roles, Users, Activity Logs UI.
- Admin Employee Work Locations UI.
- Admin Employee Schedules UI / pengelolaan jadwal.
- Employee attachment upload untuk leave/correction.
- Report Employee Attendance Report khusus per employee.
- Export Excel asli `.xlsx`; yang ada CSV.
- Notifikasi/push/email.
- Activity log UI dan audit trail detail.

### 4.7 Leave Request Approval HRD

#### Kurang

## 5. Database / Model / Infrastructure

### Sudah ada
- Tabel inti absensi/HR sudah lengkap: employees, shifts, work_locations, attendances, logs, leave, correction, summaries, approvals, activity_logs.
- Repository binding banyak model aktif sudah ada.
- Manual soft delete trait ada.
- Attachment security service ada untuk encrypted profile/attachment preview.
- `Attendance` sudah punya relasi `checkInPhoto` dan `checkOutPhoto`.
- `leave_request_details` sekarang dipakai saat HRD approve leave.

- `activity_logs` belum dipakai konsisten untuk aksi penting.

- Approval/correction/leave multi-table belum transaction-safe penuh.
- Repository pattern belum konsisten pada service Employee/HRD; masih banyak query model langsung.
