# Check Feature Todak Attendace

Tanggal audit: 2026-06-10
Metode audit: baca BRD, README, revision_feature, revision_feature_v2, route/controller/service/view/model yang relevan, dan hasil implementasi terakhir. Status ini adalah analisis kode statis + validasi syntax/cache, bukan QA manual penuh browser.

## Ringkasan Eksekutif

### Sudah ada dan relatif berfungsi
- Auth login/logout dan redirect role Admin, HRD, Employee.
- Role-based access untuk prefix Admin, HRD, Employee.
- Branding UI login dan `APP_NAME` sudah diganti ke Todak Attendace.
- Login memakai logo `icon_trans_white_1.png` dan `icon_trans_2.png` dari `public/assets/etherno/public`.
- Admin CRUD master utama: departments, positions, work locations, shifts, holidays, employees.
- Admin Parameter Aplikasi untuk set kuota cuti tahunan global tanpa input tahun.
- Employee dashboard, absensi hari ini, check-in, check-out, riwayat, detail riwayat, kalender, pengajuan izin/cuti, koreksi absensi, profil.
- HRD dashboard, monitoring absensi, peta absensi, activity calendar, belum check-in, belum check-out, terlambat, outside radius, approval pengajuan, approval koreksi, laporan harian/bulanan, export CSV, jadwal karyawan read-only, saldo cuti.
- Attachment preview signed URL untuk profile image terenkripsi.
- Foto attendance punya relasi `checkInPhoto` dan `checkOutPhoto`; HRD detail dan Employee detail menampilkan foto via signed preview URL.
- HRD detail attendance dan Employee detail attendance menampilkan peta masuk/pulang, kantor, radius, marker employee, dan garis perbandingan.

### Sudah direvisi pada implementasi terakhir
- Outside radius pada check-in/check-out wajib `work_mode_id` + `note` di backend.
- Outside radius menolak work mode `office` dan memastikan reference group `WORK_MODE`.
- Check-in terlambat mengubah `status_id` menjadi `late` jika melewati toleransi shift.
- `late_minutes` dihitung setelah toleransi shift.
- Check-out Employee menghitung `early_leave_minutes` dari shift end, termasuk overnight shift sederhana.
- HRD Belum Check-in exclude holiday, employee day off, dan approved leave.
- Employee leave request validasi overlap pending/approved.
- Employee annual leave validasi sisa `leave_balances` saat submit.
- Annual leave lintas tahun ditolak agar kuota tahunan aman.
- Leave/correction store Employee menangkap `RuntimeException` agar error bisnis tampil ke UI.
- Leave cancel dan correction cancel update status `cancelled`, bukan delete row.
- Employee correction menolak duplikasi pending untuk employee + tanggal sama.
- HRD leave approval dibungkus `DB::transaction`.
- HRD correction approval dibungkus `DB::transaction`.
- HRD leave approval membuat `leave_request_details` per tanggal.
- HRD leave approval mengurangi `leave_balances` untuk annual leave dengan row lock.
- HRD correction approval menghitung ulang `late_minutes`, `total_work_minutes`, `early_leave_minutes`, dan status present/late.
- HRD list leave request dan correction punya filter status/tanggal/departemen/keyword.
- Monthly summary generate tidak mengganti UUID existing.
- Calendar Employee memakai prioritas `attendances`, lalu `leave_request_details`, lalu fallback `leave_requests`.
- Leave approval membersihkan jam masuk/pulang/late/early/total work/need approval pada attendance leave.
- HRD query utama menyembunyikan child data milik employee soft-deleted memakai `whereHas('employee')`.
- Shift punya `applicable_days` dan `custom_days` untuk weekday/weekend/all/custom.
- New employee otomatis dibuatkan saldo cuti tahun berjalan memakai setting global.
- HRD leave balance bisa generate missing balance, adjust tambahan quota, dan set total quota.
- HRD Activity Calendar menampilkan event non-absensi: cuti tahunan, sakit, izin, hari libur.
- HRD Peta Absensi menampilkan titik check-in/out, kantor, radius, status inside/outside/need approval, dan link detail.
- Admin Parameter Aplikasi memegang setting kuota cuti tahunan global; HRD hanya generate/adjust saldo.

### Masih kurang / risiko
- Activity log table/model ada, tapi aksi penting belum konsisten dicatat ke `activity_logs`.
- `approvals` table/model ada, tapi approval leave/correction/outside radius masih langsung update tabel target, belum memakai tabel `approvals`.
- Leave/correction attachment column ada, tapi form Employee belum upload attachment pendukung.
- Attendance photo masih disimpan di public storage, walau preview sudah lewat signed route.
- Validasi outside radius di frontend belum dipastikan disable submit sampai work mode + note terisi; backend sudah wajib.
- HRD daily/monthly report ada, tapi kolom/filter belum selengkap BRD.
- HRD employee schedules masih read-only; belum ada create/update/delete.
- Admin employee detail route target BRD belum ada; hanya index/create/edit/update/delete.
- Sidebar/menu wording masih sebagian English.
- Export Excel asli `.xlsx` belum ada; yang ada CSV.
- Notifikasi/push/email belum ada.
- Password reset dan session/device management belum ada.

### 3.2 Attendance Today, Check-in, Check-out

#### Sudah ada

- Jarak ke work location dihitung.

## 5. Database / Model / Infrastructure

### Sudah ada
- Tabel inti absensi/HR lengkap: employees, shifts, work_locations, attendances, logs, leave, correction, summaries, approvals, activity_logs.
- Repository binding banyak model aktif sudah ada.
- Manual soft delete trait ada.
- Attachment security service ada untuk encrypted profile/attachment preview.
- `Attendance` punya relasi `checkInPhoto` dan `checkOutPhoto`.
- `leave_request_details` dipakai saat HRD approve leave.
- `leave_balances` dipakai untuk validasi, generate, adjust, dan pengurangan annual leave.

- `activity_logs` belum dipakai konsisten untuk aksi penting.
- Attendance photo masih public storage biasa, belum encrypted/private storage seperti profile.
- Repository pattern belum konsisten pada service Employee/HRD; masih banyak query model langsung.
