4. REVISION POINT YANG HARUS DIPERBAIKI
================================================================================

4.1 Ubah users.uuid menjadi NOT NULL
--------------------------------------------------------------------------------
Kondisi saat ini:

- users.uuid masih nullable
- ada kemungkinan data user dengan uuid NULL

Saran revisi:

UPDATE users
SET uuid = UUID()
WHERE uuid IS NULL;

ALTER TABLE users
MODIFY uuid CHAR(36) NOT NULL;

Alasan:
Agar konsisten dengan table lain yang menggunakan uuid sebagai identifier publik.

Priority: HIGH

4.2 Pastikan users.username unique
--------------------------------------------------------------------------------
Kondisi saat ini:

- users.username sudah ter-index, tetapi belum dipastikan unique

Saran revisi:

ALTER TABLE users
ADD UNIQUE KEY users_username_unique (username);

Catatan:
Sebelum menjalankan query, cek dulu apakah ada duplicate username.

Query pengecekan:

SELECT username, COUNT(*) total
FROM users
WHERE username IS NOT NULL
GROUP BY username
HAVING COUNT(*) > 1;

Priority: MEDIUM

4.3 Tambahkan unique attendance per employee per date
--------------------------------------------------------------------------------
Requirement:
Satu employee hanya boleh memiliki satu data absensi pada satu tanggal.

Saran revisi:

ALTER TABLE attendances
ADD UNIQUE KEY attendances_employee_date_unique (employee_id, attendance_date);

Sebelum menjalankan query, cek duplicate data:

SELECT employee_id, attendance_date, COUNT(*) total
FROM attendances
WHERE delete_status = 0
GROUP BY employee_id, attendance_date
HAVING COUNT(*) > 1;

Priority: HIGH

5. REVISION POINT OPSIONAL / ENHANCEMENT
================================================================================

5.1 Tambahkan leave_type_id pada leave_balances
--------------------------------------------------------------------------------
Kondisi saat ini:
leave_balances hanya memiliki employee_id dan year.

Jika sistem hanya menghitung cuti tahunan, struktur saat ini masih cukup.
Namun jika ingin lebih scalable untuk berbagai jenis cuti, tambahkan leave_type_id.

Saran revisi:

ALTER TABLE leave_balances
ADD COLUMN leave_type_id BIGINT UNSIGNED NULL AFTER employee_id,
ADD INDEX leave_balances_leave_type_id_index (leave_type_id);

ALTER TABLE leave_balances
ADD UNIQUE KEY leave_balances_employee_leave_type_year_unique
(employee_id, leave_type_id, year);

Foreign key yang disarankan:

leave_balances.leave_type_id -> references.id

Priority: OPTIONAL / MEDIUM

5.2 Tambahkan GPS accuracy meter
--------------------------------------------------------------------------------
Untuk kebutuhan validasi kualitas GPS dari device karyawan, disarankan menambahkan:

Pada table attendances:

- check_in_gps_accuracy_meter
- check_out_gps_accuracy_meter

Pada table attendance_logs:

- gps_accuracy_meter

Saran revisi:

ALTER TABLE attendances
ADD COLUMN check_in_gps_accuracy_meter DECIMAL(10,2) NULL AFTER check_in_distance_meter,
ADD COLUMN check_out_gps_accuracy_meter DECIMAL(10,2) NULL AFTER check_out_distance_meter;

ALTER TABLE attendance_logs
ADD COLUMN gps_accuracy_meter DECIMAL(10,2) NULL AFTER distance_meter;

Priority: OPTIONAL / MEDIUM

5.3 Tambahkan source pada attendance_logs
--------------------------------------------------------------------------------
Untuk mengetahui absensi berasal dari mana, tambahkan source.

Contoh value:

- web
- mobile
- admin_correction
- system

Saran revisi:

ALTER TABLE attendance_logs
ADD COLUMN source VARCHAR(50) NULL AFTER user_agent;

Priority: OPTIONAL / LOW

5.4 Validasi latitude dan longitude pada work_locations
--------------------------------------------------------------------------------
Kondisi saat ini:
work_locations.latitude dan work_locations.longitude masih nullable.

Jika semua work location adalah kantor fisik, sebaiknya dibuat NOT NULL.
Namun jika ada lokasi fleksibel seperti WFH atau Remote, boleh tetap nullable.

Rekomendasi:
Untuk tahap awal, biarkan nullable di database, tetapi validasi di level aplikasi:

- jika is_active = true dan tipe lokasi adalah kantor, latitude wajib
- jika is_active = true dan tipe lokasi adalah kantor, longitude wajib
- radius_meter wajib lebih dari 0

Priority: OPTIONAL / LOW
